Introduction

It's given me a lot of joy and was one of the driving forces in my introduction to computer programming. It's a fairly niche idea outside of programmers, so I feel the need to expel a little bit of my excitement towards it on this webpage.

Esoteric Programming Languages

Esoteric programming languages are a class of languages that aren't built necessarily because they have good features, offer high performance, or even technically function. Esoteric Languages (Esolangs) are conceptualized and built to serve another purpose. These can include artistic experiments, jokes, proofs of concept, and even rehashes of other esolangs.

Some notable examples of esolangs are Shakespeare, Chef, Thue, Befunge, LOLCODE, and Whitespace.

Enter: Brainfuck

Brainfuck is an esolang that was proposed by a guy named Urban MÃ¼ller in the early 90's. It's incredibly minimal with only 8 operators to get all your programming done. It's a descendent and derivative1 of P'', another esolang. Brainfuck is technically turing complete, but actually accomplishing anything in it is incredibly difficult. As the name may imply, brainfuck is quite the brainfuck. It's less programming and more gluing operators together until something happens2.

The language has a bit of a cult following. With outstanding projects like a mandlebrot set generator, a CPU, and endless code golf implementations, it's hard to understate just how fun this language has proven itself to be. It is also the subject of many code golf competitions.3

Brainfuck works by moving a pointer forward and backward in memory. Brainfuck's operators revolve around moving this pointer and changing the value of the data it points to.

Brainfuck's first six operators:
+ Increment cell
- Decrement cell
> Shift pointer right
< Shift pointer left
. Output current cell in ASCII
, Input to current cell

These are pretty simple. Think of it as a number line but instead of digits, you have data. For example, say we wanted to write out the numbers 4, 8, and 16 in memory. We would do it like this:
++++       ;Starting at memory location 0, increment by 4
>          ; Move to the next location in memory
++++ ++++  ; Increment location by 8
>          ; Move to the next location in memory
++++ ++++ ++++ ++++    ; Increment cell by 16

This leaves us with three non-zero cells containing the numbers [4, 8, 16].
